¿Y ahora qué?

Modelo de Objetos de PHP

Objecto

Tipo de dato compuesto que permite agrupar en una misma estructura funciones y variables (propiedades).

Clases

Objeto aún no creado. No es lo mismo el tipo de dato string que una variable de tipo string. Lo mismo sucede con las clases y objetos respectivamente.

Clases abstractas

Clases de las que no se puede crear ningún objeto. ¿Para qué sirven entonces? Pueden crearse subclases de ésta. De las subclases sí se pueden crear objetos.

Interfaces

Son convenios (acuerdos) a la hora de qué funciones debe tener una clase.

Traits

Clases parciales (herencia múltiple).

Espacios de nombres

¿Qué ocurre si utilizas una biblioteca que declara nombres de funciones que tú ya habías usado? Produciría un error: dos funciones no pueden tener el mismo nombre.

Los espacios de nombres (namespaces) son un mecanismo que evita estos problemas.

Extensiones

¿Qué sistema podemos usar para compartir bibliotecas de funciones? Podemos hacerlo a mano, o por medio de extensiones.

PEAR es la propuesta de PHP.

phar es un mecanismo de PHP para encapsular toda una aplicación o framework en un único fichero, para facilitar su instalación, distribución e instalación.

Composer es un repositorio de proyectos que podemos incluir en el nuestro. Utiliza phar como mecanismo de distribución.

Expresiones regulares

Son un mecanismo muy potente (¡y complejo!) que permite analizar cadenas de texto para comprobar si cumplen un determinado patrón o para extraer partes concretas del texto que, a priori, no podemos saber dónde está.

Por ejemplo, ¿cómo nos aseguramos de que un usuario ha introducido correctamente un teléfono? ¡Hay muchos formatos!

SQL

Es un lenguaje de programación de alto nivel para trabajar con bases de datos.

Los principales SGBDR (Sistemas de Gestión de Bases de Datos Relacionales) permiten trabajar con SQL. Por ejemplo, el famosísimo MySQL.

PHP incorpora bibliotecas para trabajar con MySQL, PostgreSQL, Microsoft SQL Server, SQLite, etc.

XML y JSON

XML es un lenguaje (eXtensible Markup Language) para almacenar, estructurar e intercambiar información. Se utiliza a menudo al comunicar una aplicación con un servicio externo (Twitter, Facebook). La 'X' de AJAX es de XML: Asynchronous Javascript And XML.

JSON es otro modelo (JavaScript Object Notation) con las mismas funciones que XML, pero mucho más sencillo.

Los servicios externos más populares suelen permitir trabajar con ellos tanto con XML como con JSON.

PHP incorpora numerosas herramientas para trabajar tanto con XML como con JSON.

API REST

¿Y por qué no crear tu propio servicio para que otras aplicaciones consuman datos de ella? Para ello puedes utilizar el patrón REST para crear una API. Una API es una Interfaz Programación de Aplicaciones. Es decir: similar a una biblioteca de funciones.
