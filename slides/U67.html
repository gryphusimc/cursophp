Servidores web

Existen muchos servidores de páginas web. Son los encargados de esperar peticiones y de responder a éstas.

Hemos utilizado un servidor muy sencillo: el servidor interno de PHP. Pero este servidor está pensado solo como entorno de desarrollo.

Para entornos más complejos, de pruebas y de producción, necesitaremos otro servidor más adecuado.

Características de servidores web

  • Sirven ficheros estáticos y dinámicos.
  • Soporte para SSL/TLS.
  • Hosts virtuales.
  • Compresión.
  • Sistema de logs.
  • Control de accesos.
  • Reescritura de URL.
  • Balanceo de carga.
  • FastCGI (Common Gateway Interface).

Servidor web Apache

El más popular es, sin duda, Apache. Aunque su uso se está viendo reducido con la llegada de servidores ligeros como nginx o lighttpd. Hay páginas que utilizan los dos; Apache para contenido dinámico (PHP) y otro para contenido estático (imágenes, CSS, JavaScript).

Apache soporta un gran número de extensiones

Por qué Apache

Fue el servidor que hizo crecer a Internet. Al ser tan popular, existe muchísima documentación y extensiones para ampliarlo. Además, es el que incorporan la mayoría de empresas de alojamiento web en los planes Linux (en Windows se suele utilizar IIS, el servidor de Microsoft).

PHP y Apache

PHP contiene funciones específicas para trabajar con el servidor web Apache. Por ejemplo:

string apache_get_version ( void )

Nos devuelve la versión de Apache sobre la que estamos trabajando.

Mientras que:

array apache_get_modules ( void )

Nos indica qué modulos tenemos disponibles en nuestra instalación de Apache.

.htaccess

Desde Apache se puede establecer la configuración global del servidor. Pero también configuraciones por hosts virtuales (dominios). Y, además, permite que el usuario aplique configuraciones a nivel de directorio. Esto último se realiza añadiendo el fichero .htaccess a la carpeta que se desea que se aplique la configuración.

Esta configuración a su vez, y de manera predeterminada, afectará a todas sus subcarpetas.

Control de acceso

Podemos proteger una carpeta directamente desde HTTP, sin necesidad de cookies ni sesiones. En el fichero .htaccess ponemos lo siguiente:


AuthUserFile /ruta/absoluta/al/fichero/.htpasswd
AuthName Mensaje_de_autenticación
AuthType Basic

require valid-user

Para conocer la ruta absoluta podemos añadir este código PHP a un fichero en la misma carpeta:


<?php die($_SERVER["DOCUMENT_ROOT"]);

¿Y cuál es el fichero .htpasswd?

.htpasswd

Es el fichero que contiene la lista de usuarios y contraseñas codificadas.

Debe estar en una ruta inaccesible mediante navegación web.

El formato ya lo conocemos: una línea por usuario. Y para cada usuario, se pone el nombre y la contraseña codificada, y separados por :. Por ejemplo:


usuario:$apr1$svGZNl0h$W.fukM5TAByk4.5iwI5dI0

Podemos generar las contraseñas de varias formas. Por ejemplo, usando un servicio web, o desde la consola (si tenemos Apache):


htpasswd -nb usuario password

Reescritura de URL

Permite que URLs que enlazan a rutas que no existan sean gestionadas por un determinado archivo. Este archivo podría recibir la ruta inexistente en un parámetro GET, o consultando $_SERVER['REQUEST_URI'].

Por ejemplo, podemos tener esta URL:


http://www.mipagina.com/noticias/nueva-version-de-php

E indicar que, si la URL no existe, la atienda el fichero index.php.

De esta forma:

  • Tenemos URLs más amigables.
  • Las URL son semánticas: mejora el posicionamiento.
  • Una URL diferente para cada recurso

Reescritura en Apache

En Apache se utiliza el módulo mod_rewrite. Prueba a añadir este código al fichero .htaccess, crea un fichero index.php y carga cualquier URL que no exista:


<IfModule mod_rewrite.c>
RewriteEngine On
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]
</IfModule>

Copia este código al fichero index.php y accede a una URL que no exista:


<pre>
<?php
print_r($_SERVER['REQUEST_URI']);
?>
</pre>
