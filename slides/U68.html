Alojamientos web

Por Jesús Ruiz-Ayúcar / @chuso

Compañías

Para que una web esté disponible en cualquier momento, debe estar instalada en un servidor web operativo las 24 horas.

Además, debe tener una alta velocidad de subida de datos, que garantice picos en los que muchos usuarios accedan simultaneamente a la web.

Para ofrecer calidad de conexión (entre otras muchas ventajas), surgen las compañías de hosting.

En qué fijarnos

  • Soporte de PHP — Una versión de PHP reciente suele ser una buena señal. ¿Permite adaptar php.ini?
  • Bases de datos — Seguramente necesitemos MySQL o MariaDB. Quizás MongoDB.
  • Acceso — ¿Permite SSH? ¿O solamente FTP?
  • Espacio — Para datos, bases de datos y correos.
  • Tráfico — ¿Ilimitado o limitado?
  • Dominios — ¿Permite un dominio o varios?

Tipos de alojamientos

Existen muchos planes distintos y diferentes. Los más populares (por orden de más barato a más caro) son los siguientes:

  • Alojamiento compartido — Compartes máquina (CPU, RAM, disco) con otros dominios, normalmente cientos.

    • De un dominio — solo puedes alojar un dominio.
    • Multidominio — puedes alojar muchos dominios.
  • Servidor virtual — Dispones de una máquina virtual. Puedes hacer con ella lo que quieras. Comparte máquina física con otros servidores virtuales.
  • Servidor dedicado — Dispones de una máquina física solamente para ti.

Acceso FTP y SFTP

Necesitamos poder enviar los ficheros entre nuestro equipo y el servidor (para instalar la web). Existen varios protocolos:

  • FTP lo utilizamos para transferir ficheros a un servidor remoto. Todo el tráfico viaja sin cifrar, por lo que es poco seguro.
  • FTPS es una actualización de FTP para hacerlo seguro.
  • SFTP protocolo similar, con el tráfico cifrado mediante SSH.


Para usarlo, necesitaremos un cliente FTP (suelen englobar FTP, FTPS y SFTP). Existen muchos. De hecho hay editores de código que incluyen acceso remoto mediante FTP para editar en remoto. Un cliente muy sencillo es el famoso Filezilla.

Acceso SSH

Una característica muy interesante a la hora de contratar un alojamiento es el acceso mediante SSH.

Si te sientes cómodo trabajando con la consola, necesitarás SSH. Permite iniciar sesión en un servidor remoto mediante la consola. Puedes navegar por directorios, comprimir y descomprimir copias de seguridad, buscar cadenas de texto, etc.

Dominios

Los servidores web, como Apache, crean un host virtual para alojar dominos (o subdominios). Cuando queramos crear una página, necesitaremos uno para poder identificarla. Las propias compañías de hosting también venden dominios. Puedes tener un dominio contratado con una compañía y el alojamiento con otra. Pero si es tu primera vez, es recomendable hacerlo todo en la misma empresa. De esa forma no tienes que preocuparte por nada; simplemente funciona.

Mapeo local de dominios

Un nombre de dominio está vinculado a una dirección IP, normalmente la del servidor web. Esta información puede tardar bastante tiempo (unas 24-48 horas) en hacerse efectiva desde que se registra o actualiza un dominio.

Para poder continuar trabajando, podremos forzar el que en nuestro equipo un determinado dominio apunte a la IP que nosotros indiquemos.

También nos es útil este forzado si estamos trabajando en una web local y queremos acceder a ella usando su nombre de dominio. Vincularíamos ese nombre de dominio a la IP local: 127.0.0.1.

/etc/hosts

Para mapear (enlazar) localmente (en nuestro equipo) un dominio, debemos editar el fichero /etc/hosts. Es necesario editarlo con permisos de Administrador, tanto en Windows, como en Linux/Mac.

En Linux/Mac está en /etc/hosts. En Windows está en C:\Windows\System32\drivers\etc\hosts.

Y tiene esta estructura:


#IP dominio
127.0.0.1        localhost
173.194.34.152   google.es
