Formularios HTML

Permiten enviar datos a una URL. Implican una recarga de página (síncrono). Si no quisiéramos recargar la página, deberíamos usar JavaScript (AJAX, asíncrono).

Ejemplo de formulario HTML:


<form method="get" action="index.php">
	<!-- Contenido del formulario -->
</form>

En method podremos poner get (predeterminado) o post. En action indicamos la URL a la que enviamos el formulario (por omisión, se envía a la misma URL que lo contiene).

Elementos de un formulario

Dentro de un formulario añadimos elementos donde el usuario puede introducir o seleccionar un valor, por ejemplo:


<form method="get" action="index.php">
    <div>
        <input type="text" name="nombre" placeholder="Introduce tu nombre" />
        <br />
        <button>Enviar</button>
    </div>
</form>

Tenemos un elemento, input, de tipo text.

Hay campos de texto, de texto multilínea, casillas de verificación, elementos desplegables, etc.

El valor de estos campos lo enviará el formulario al documento php que indiquemos.

GET

Como vimos, es un método HTTP que solicita un recurso. Es el único método HTTP que hemos usado hasta ahora.

Podemos enviar un formulario usando este método HTTP. Los valores se añadirán a la URL con la siguiente sintaxis:

fichero.php?param1=val1&param2=val2...

Por ejemplo:


http://localhost:8080/index.php?nombre=Juan+Sin+Miedo

Recuerda parse_url(): analizaba una URL y devolvía un array. En la clave query estaban los parámetros GET. Puedes usar parse_str() para extraerlos o usar $_GET.

$_GET

Es una variable predefinida que almacena un array donde cada clave es un valor recibido por GET.

Por ejemplo, podríamos saludar al usuario:


<?php if (empty($_GET['nombre'])): ?>
<form method="get">
    <div>
        <input type="text" name="nombre" placeholder="Introduce tu nombre" />
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: ?>
Bienvenido, <?=$_GET['nombre']?>
<?php endif; ?>

El valor del atributo name del campo de entrada es el que se usa como clave de $_GET.

POST

Se trata de otro método HTTP: envía datos a un recurso. Lo genera un formulario que utilice este método.

A diferencia de GET, POST no envía los datos en la propia URL, sino que codifica toda la información en la cabecera de la petición.

Por tanto, con POST la URL queda limpia.

$_POST

Es una variable predefinida que almacena un array donde cada clave es un valor recibido por POST.

Revisamos el ejemplo anterior:


<?php if (empty($_POST['nombre'])): ?>
<form method="post">
    <div>
        <input type="text" name="nombre" placeholder="Introduce tu nombre" />
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: ?>
Bienvenido, <?=$_POST['nombre']?>
<?php endif; ?>

El valor del atributo name del campo de entrada es el que se usa como clave de $_POST.

Elementos de texto

Ya hemos visto un ejemplo de campo de texto. Con la llegada de HTML5 se incluyeron muchos otros elementos basados en texto:

  • email
  • number
  • search
  • tel
  • url

Con todos ellos recibiremos en $_GET o $_POST el valor introducido por el usuario. Siempre se usará como clave el valor del atributo name del elemento de entrada.

Área de texto

Se trata de un campo de texto multilinea:


<form method="post">
    <div>
        <textarea name="descripcion" ><?=$_POST['descripcion']?></textarea>
        <br />
        <button>Enviar</button>
    </div>
</form>

Menú desplegable

Se recibe el valor del elemento seleccionado.


<?php if (empty($_POST['navegador'])): ?>
<form method="post">
    <div>
        <select name="navegador" >
        	<option value="">Seleccione su navegador favorito</option>
        	<option value="chrome">Google Chrome</option>
        	<option value="firefox">Mozilla Firefox</option>
        	<option value="ie">Internet Explorer</option>
        	<option value="safari">Safari</option>
        </select>
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: ?>
Has seleccionado <?=$_POST['navegador']?>
<?php endif; ?>

Botones radio

Se recibe el valor del elemento seleccionado.


<?php if (empty($_POST['sexo'])): ?>
<form method="post">
    <div>
    	<input type="radio" name="sexo" value="M" id="sexo_M" />
    	<label for="sexo_M">Masculino</label><br />
    	<input type="radio" name="sexo" value="F" id="sexo_F" />
    	<label for="sexo_F">Femenino</label><br />
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: ?>
Tu sexo es: <?=$_POST['sexo']?>
<?php endif; ?>

Casilla de verificación

Se envía su valor si se selecciona. Si no, no se envía ni su clave:


<?php if ($_SERVER['REQUEST_METHOD'] != "POST"): ?>
<form method="post">
    <div>
    	<input type="checkbox" name="condiciones" value="1" id="condiciones" />
    	<label for="condiciones">Acepto las condiciones de uso</label><br />
    	<input type="checkbox" name="promociones" id="promociones" />
    	<label for="promociones">Autorizo el envío de mails promocionales</label>
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: print_r($_POST); endif; ?>

Wrapper input://

Es un wrapper (fichero) que contiene los datos de entreda de un script PHP cargado mediante POST.

Puedes consultar su contenido así:


<?php $postdata = file_get_contents("php://input"); ?>

Aunque no suele ser práctico; ya tenemos $_POST.

Arrays y formularios

Puedes usar sintaxis de arrays PHP al asignar nombres a los elementos del formulario HTML. PHP, cuando lo reciba, lo convertirá a un array:


<?php if (empty($_POST['usr']['nombre'])): ?>
<form method="post">
    <div>
        <input type="text" name="usr[nombre]" placeholder="Tu nombre" />
        <input type="text" name="usr[apellidos]" placeholder="Tus apellidos" />
        <br />
        <button>Enviar</button>
    </div>
</form>
<?php else: ?>
Bienvenido, <?=$_POST['usr']['nombre']?> <?=$_POST['usr']['apellidos']?>
<?php endif; ?>
